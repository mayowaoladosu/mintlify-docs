## Week of June 20-26, 2026

This week focused on hardening the LayerRail console deployment path, improving Azure-backed resource reliability, and tightening the user-facing console experience across Game VPS, PostgreSQL, Kubernetes, load balancers, Intercom support, and regional availability.

### New

- **Intercom support in the console.** Integrated the Intercom messenger and refactored project sidebar navigation so users can reach support from inside the console without leaving their project flow.
- **Admin force-destroy controls.** Added admin-side force destroy controls for stuck resources so operations can clear orphaned or stalled infrastructure faster.
- **Automatic database migrations on startup.** Added web startup migration handling and deployment workflow migration steps so production services can apply schema changes during deploys more reliably.
- **More Azure locations.** Added South Africa North and UK South to the Azure-backed console location list.

### Updated

- **Azure Game VPS provisioning.** Retried Azure Game VPS network convergence so NIC, subnet, and virtual network provisioning waits more safely for Azure resources to reach a usable state.
- **Azure region catalog.** Removed ineligible Azure locations, including West Europe and North Europe, to keep the console aligned with currently supported subscription capacity.
- **Repository references.** Updated repository URL references to the LayerRail GitHub organization.
- **CI and Azure deployment workflow.** Updated the Azure deployment path with migration container fixes, mandatory encryption/session secrets, Linux container OS selection, and corrected container command syntax.

### Fixed

- **PostgreSQL failure visibility.** PostgreSQL VM failure states now bubble up properly, and the loader behavior was fixed on Windows.
- **Load balancer nil-location crashes.** Guarded `LoadBalancer#path` and `LoadBalancer#display_location` against missing location data so the console does not crash on incomplete or transitional records.
- **PostgreSQL cron setup.** Ensured the `pg_cron` package is installed and `include_dir` is active universally.
- **Procfile formatting.** Fixed web process formatting so runtime process startup is cleaner.


## Week of June 13-19, 2026

This week focused on improving the Deploy platform, enhancing Kubernetes node joining and cleanup operations, extending Cloudflare AI integrations, and refining deployment, billing, and provider infrastructure management.

### New

- **LayerRail Deploy platform upgrade.** Major rework of the deploy platform including building deploy apps as container images, expanding build packs, and a new deploy-specific GitHub connection flow.
- **Unified AI models.** Added Cloudflare unified AI models and expanded the Cloudflare AI model catalog.
- **Intercom support.** Added Intercom support triggers.
- **PgBouncer management.** Implemented simple PgBouncer systemd service configuration handling for production databases.
- **Deploy schema and imports.** Added cached schema exposure for new deploy fields and reworked deploy import and app domain flows.
- **Multiple Game VPS subscriptions.** Added the ability to allow multiple Game VPS subscriptions.

### Updated

- **Load balancer UI.** Updated branding and refreshed the load balancer waiting page.
- **Deploy and domain workflows.** Rebuilt deploy and domain checkout flows, and lowered core TLD pricing.
- **Domain TLD sync.** Synced the domain TLD catalog directly from the registrar and cleaned up the domain management UI.
- **AI playground limits.** Tightened the AI playground model list and defined default max output tokens for responses.
- **Resource deletion.** Ensured billing stops immediately upon resource deletion.
- **Kubernetes operations.** Switched Azure Kubernetes joins to use private endpoints and local kubeconfigs, separating Kubernetes nftables from load balancers. Added steps to set node addresses and accept ready nodes post-join.
- **Azure infrastructure.** Updated Azure to use stable data disk paths and retry on transient VM creation failures. Allowed using supported Azure disk delete APIs.

### Fixed

- **Kubernetes cleanup and scaling.** Fixed Kubernetes deletion retry stalls, node pool scale tracking, and load balancer cleanup on deletion. Addressed stuck Kubernetes creation issues.
- **AI playground stability.** Fixed Cloudflare AI catalog presentation, Azure cleanup, and various AI playground errors.
- **Service teardowns.** Fixed broader service cleanup processes to ensure consistent resource deletion.
- **Checkout and reconciliation.** Improved Game VPS checkout rescue scripts, handling of integer checkout amounts, and reconciliation diagnostics. Broadened Polar checkout reconciliation fields.
- **Deploy UI rendering.** Fixed deploy GitHub import rendering and deploy table rendering issues.
- **Provider configurations.** Fixed Azure VM cleanup, Game VPS plan ladder, and Game VPS detail routing.
- **Timeout handling.** Handled NameSilo timeout errors cleanly.


## Week of June 7-12, 2026

This week moved LayerRail from a compute console into a broader cloud platform surface: AI app endpoints, domain registration workflows, Azure-backed infrastructure, prepaid Game VPS subscriptions, and cleaner production operations.

### New

- **AI Apps layer.** Added a managed AI app surface for creating product-facing AI endpoints with prompts, model selection, API access, and project-level association.
- **App endpoint pages.** Added endpoint detail pages with usage examples, API URLs, and responsive layouts for developers integrating AI endpoints into their own products.
- **Domain registration foundation.** Added domain search, cart, contact profiles, registration flow, DNS zone creation, billing reconciliation, and domain detail views.
- **Domain registrar phase two and three workflows.** Added transfer flows, renewal handling, DNSSEC direction, domain forwarding, project attachment, abuse controls, email alert foundations, API-oriented domain tools, bulk workflows, sub-account/team controls, domain-to-app deployment hooks, bundle concepts, and ICANN evaluation notes.
- **Admin controls for domains.** Added pricing controls, TLD enable/disable controls, and admin-facing domain management hooks.
- **Azure compute provider.** Added the first Azure-backed compute provider implementation for new infrastructure deployments.
- **Azure deployment workflow.** Added Azure production deployment support, including worker health entrypoints and namespace/autoload fixes.
- **Azure-backed Game VPS plans.** Added Windows Game VPS plans backed by Azure infrastructure.
- **Prepaid Game VPS checkout.** Added monthly prepaid Game VPS checkout through Polar, with per-plan product mapping.
- **Windows Game VPS access fields.** Game VPS creation now collects a Windows username and password, validates them, stores the password securely, and passes credentials into provisioning.
- **RDP profile downloads.** Game VPS detail pages now provide an `.rdp` download once the server has an IP address.

### Updated

- **Lower VM pricing presentation.** Adjusted Azure-backed plan pricing to keep entry plans approachable while Azure becomes the default provider for new deployments.
- **Compute catalog alignment.** Updated the compute catalog to match available Azure SKUs, images, and regions more closely.
- **Longer init scripts.** Increased init script length limits so larger bootstrap scripts can be used during VM creation.
- **Cleaner Game VPS labels.** Removed provider-facing labels from Game VPS locations and image choices so the console presents LayerRail product names instead of upstream provider names.
- **Game VPS icons.** Replaced the rocket icon with a controller-style icon across the sidebar, dashboard card, and Game VPS empty state.
- **Streak badge polish.** Tightened the dashboard login streak badge so it is closer to a compact profile-style badge.
- **Brand assets.** Added new LayerRail branding assets, including banner and logo package updates.
- **Cloud site study cleanup.** Fixed product naming in the LayerRail cloud website study.

### Fixed

- **Dashboard crash from missing icon names.** Fixed invalid dashboard icons that caused 500 errors on project dashboard pages.
- **AI app project associations.** Fixed AI app ownership and project association behavior.
- **AI app endpoint crashes.** Fixed app endpoint show page errors and improved responsive behavior.
- **Domain dashboard stability.** Fixed domain dashboard errors and schema-cache issues after domain tables changed.
- **Domain transfer icon.** Fixed the transfer icon used in the domain UI.
- **Admin noise.** Hid active page noise from the admin home screen so production operations are easier to scan.
- **Linode PostgreSQL fallback.** Fixed PostgreSQL size fallback behavior for existing Linode-backed services.
- **Linode provisioning hardening.** Hardened service provisioning and improved failure reporting for existing Linode-backed Kubernetes, PostgreSQL, and load balancer paths.
- **Game VPS checkout schema cache.** Fixed stale schema cache behavior that could break Game VPS checkout after deployment.

## Week of June 1–6, 2026

A focused week on provisioning reliability across Kubernetes, PostgreSQL, and load balancers, plus billing polish and a cleaner AI inference experience.

### New

- **Sentry error reporting in the console.** Production errors are now captured automatically so issues can be diagnosed and resolved faster.
- **Native Cloudflare AI models support.** LayerRail now exposes Workers AI tasks beyond chat and embeddings, including image generation, speech transcription, text-to-speech, translation, summarization, reranking, classification, object detection, and image-to-text through the console playground and `/ai/v1/run`.
- **Polar metered usage portal.** View metered usage directly from the billing portal for clearer visibility into consumption-based charges.
- **LayerRail-branded invoices.** Invoice PDFs now ship with LayerRail branding for a consistent billing experience.

### Updated

- **Cleaner [AI inference](/ai-inference/overview) surface.** A refreshed catalog and [playground](/ai-inference/playground) make model discovery and testing easier.
- **Redesigned load balancer empty state.** New [load balancers](/networking/load-balancer) now show an intentional waiting page before traffic is attached.
- **Longer Kubernetes provisioning windows.** Extended [cluster](/managed-kubernetes/overview) node provisioning deadlines accommodate slower production boots.
- **Polar verification charge set to $1.** The verification hold is now a flat one-dollar amount.

### Fixed

- **Login streaks.** Authentication no longer fails when streak data is unavailable or migrating.
- **Linode service bootstrap.** Resolved deadlocks and tightened the provisioning lifecycle so [virtual machines](/virtual-machines/overview) move through creation more consistently.
- **PostgreSQL provisioning.** Hardened initialization checks, fixed empty database provisioning deadlines, and added clearer bootstrap progress logging. See [Managed PostgreSQL](/managed-postgresql/overview).
- **Kubernetes networking and joins.** Fixed the worker join endpoint used during node registration and corrected private networking behavior on Linode.
- **Load balancer health checks.** Linode load balancer health checks now report correctly.
- **DNS provisioning.** Resolved Linode DNS provisioning flow errors.
- **Certificate cleanup.** Malformed ACME revoke responses are now handled gracefully.
- **Service firewalls.** Control-plane SSH is now allowed during provisioning.
- **Invoice finalization.** Polar invoice finalization is now visible end-to-end.
